## 一、这套系统是什么，能做什么

一个纯免费、部署在网页上的个人投资信息助手，打开网址随时看，不需要你告诉它持仓，
只根据公开数据做技术面提示，辅助你自己判断买卖时机。一共 6 个功能模块：

| 标签页 | 功能 |
|---|---|
| 👀 加密货币关注 | 输入你关心的币（支持直接打代号如 xrp、sui），显示现价、涨跌幅，并基于 RSI、均线给出"关注买入区/偏空/中性"提示 + 具体理由，同时显示该币前10大地址的持仓集中度（巨鲸筹码信号） |
| 💱 外汇关注 | 同样的技术面打分逻辑用在外汇货币对上（如 USD/JPY、EUR/USD），数据源是欧洲央行免费汇率 |
| 🚀 新币动量扫描 | 挖掘刚启动、涨幅异常的新币，按"涨幅+换手强度+买卖力量"综合评分排序，附带项目背景简介、社交链接、合约安全检测 |
| 🐋 巨鲸监控 | 输入你关注的巨鲸钱包地址，查看最近的代币转入/转出活动流水（客观展示，不自动判断买卖方向，避免误导） |
| 📈 信号回测 | 把现在用的打分规则套用到过去一年的真实历史价格上，统计"这个信号出现后未来N天真实涨跌概率"，验证规则靠不靠谱 |
| 📰 每日报告 | 一键把当前几个模块已加载的数据拼成一份文字摘要，快速过一遍全貌 |

顶部还有一个市场概览：BTC市占率 + 恐慌贪婪指数，帮你判断当前大盘情绪。

全部数据源都是免费公开API（CoinGecko、DexScreener、GoPlus Security、欧洲央行、
alternative.me），除了"巨鲸监控"需要你自己申请一个免费的 Etherscan API Key 之外，
其余模块打开就能用，不需要任何配置。

**再次强调**：所有提示都是基于公开技术指标的规则打分，不是预测，更不构成投资建议，
最终买卖决策需要你自己判断。

## 二、部署教程（全程免费，大概20分钟）

### 第一步：把代码传到 GitHub

1. 没有 GitHub 账号先注册：https://github.com/signup（免费）
2. 新建一个仓库，比如叫 `crypto-monitor`，建议选 **Private**（私有）
3. 把这个代码包里的文件全部上传：`core.py`、`streamlit_app.py`、`app.py`、
   `requirements.txt`、`.gitignore`
   - 网页上传最简单：进入仓库页面点 `Add file → Upload files`，把文件拖进去，点 `Commit changes`
   - 如果之前已经建过仓库，这次只需要把这几个文件重新上传一遍覆盖旧版本即可

### 第二步：部署成网页（二选一）

**方案A：Streamlit Community Cloud**（如果你之前已经用 GitHub 登录成功过，直接用这个）

1. 打开 https://share.streamlit.io，用 GitHub 账号登录
2. 点 `New app` → 选择仓库 `crypto-monitor` → 分支选 `main`
3. Main file path 填 `streamlit_app.py`
4. 点 `Deploy`，等 1-2 分钟

**方案B：Hugging Face Spaces**（如果方案A登录一直报错，用这个代替，不走GitHub登录）

1. 打开 https://huggingface.co/join 免费注册（邮箱即可）
2. 打开 https://huggingface.co/new-space 创建 Space：
   SDK 选 Streamlit（没有就选 Docker → 下一步选 Streamlit 模板），Hardware 选免费 CPU
3. 上传 `app.py`、`core.py`、`requirements.txt` 三个文件（HF 默认识别的入口文件名是 `app.py`）
4. 等 1-2 分钟自动构建

### 第三步：以后随时打开

部署成功后你会拿到一个固定网址，形如：
- Streamlit Cloud: `https://你的名字-crypto-monitor-xxxxx.streamlit.app`
- Hugging Face: `https://你的用户名-crypto-monitor.hf.space`

**建议做的事**：
- 把这个网址加到手机浏览器主屏幕（iOS：分享→添加到主屏幕；Android：菜单→添加到主屏幕），
  以后像打开App一样一键打开，不用记网址
- 长时间没访问会"休眠"，重新打开需要等十几秒唤醒，这是免费额度的正常现象，等一下就好

## 六次更新说明（巨鲸每日报告 + 币种简写识别 + 修复获取失败问题）

### 1. 巨鲸监控改成"今日报告"
不再是一条条转入转出流水，而是：
- **按代币聚合今天的净流入/净流出**（比如"净流入15 ETH，净流出5万USDC"），一眼看出今天整体在囤还是在出
- **主要交易对手方分析**：查一下今天交易最频繁的几个对手方，是合约还是普通钱包、
  合约叫什么名字（这是从Etherscan查到的客观事实，不是猜的）。如果对手方是DEX路由类合约，
  会提示"可能是在做链上swap"；如果是普通钱包，会提示"更像是资金归集/分仓/转账"
- 详细的原始流水收进了"查看今天的完整流水明细"折叠框，每笔都带了区块浏览器链接方便你核实
- **老实说清楚一点**：解读部分是基于客观数据的合理推测方向，不是确定结论——链上数据本身
  不包含"意图"，没有任何免费数据源能100%准确告诉你"这笔操作背后是想干嘛"，需要你自己
  点交易链接核实细节再下判断

### 2. 加密货币现在真的只填简写就行
之前 xrp、sui 偶尔查不到，排查下来主要是两个原因，这次一起解决了：
- 内置了一张常见币种代号对照表（btc/eth/xrp/sol/sui/bnb/doge等60多个主流币），
  命中的话直接用，不用再调用搜索接口，基本不会再出现"找不到"的情况
- 对不在表里的生僻币种，加了自动重试和更准确的报错提示（能分清"真的没这个币"
  和"接口被限流了"这两种情况）

### 3. 强烈建议加一个免费的 CoinGecko Demo API Key
之前经常"获取数据失败"，根本原因是CoinGecko完全免费不注册的公开接口只有
**5~15次/分钟**的限速，非常容易触发。免费注册一个Demo Key能稳定在30次/分钟：
1. 打开 https://www.coingecko.com/en/api/pricing → 点 `Create Free Account`
2. 注册登录后进 Developer's Dashboard → API Keys → `+ Add New Key`
3. 把生成的Key填到侧边栏最上面的"🦎 CoinGecko Demo Key"栏，点保存

这个Key全程免费，不需要绑卡，加了之后"关注币种""信号回测"这些模块会明显更稳定。

## 七次更新说明（技术面多因子升级 + 衍生品杠杆情绪层）

参考了你贴的那份"多因子评分系统"文档，把里面**真正能免费做到位**的两块做了进来：

### 1. 技术面从"RSI+简单均线"升级成多因子打分
- **EMA20/50/200 多周期结构**：不再是简单的"现价>MA7>MA25"，现在会判断"现价<EMA200
  但现价>EMA20>EMA50"这种"熊市末期恢复"的结构，以及标准的多头/空头排列
- **MACD柱状图转折**：由负转正/由正转负，以及"柱子在缩小=动能减弱"这种细节
- **RSI背离检测**：价格创新低但RSI没跟着创新低（底背离，卖压减弱）、价格创新高但RSI
  没跟着创新高（顶背离，上涨动能减弱），这比单纯"RSI<30就是超卖"更接近专业交易员的判断方式
- 因为因子变多了，评分的阈值也相应调整了（现在要更多因子共振才会给出"关注买入区"这种
  强信号），"信号回测"那个标签页可以直接验证这套新规则的历史胜率

### 2. 新增"衍生品层"：资金费率 + 未平仓合约
数据源是 **Binance合约公开API，完全免费不需要key**。这是加密货币独有的、反映短期杠杆
情绪是否过热的硬指标：
- 资金费率过高 = 多头拥挤，警惕挤仓回调；过低（负值）= 空头拥挤，可能挤空反弹
- 未平仓合约变化 = 杠杆仓位是在加仓还是在减仓
- 两者结合价格走势，能识别"价格涨+持仓涨+费率涨"这种历史上常见的杠杆过热顶部信号，
  以及"价格跌+持仓降+费率回落"这种杠杆出清后的潜在机会窗口
- 只覆盖 Binance 有合约上架的币种，小市值新币可能没有，会显示"未覆盖"

侧边栏新增了对应的开关，可以选择要不要查这两项（多两次请求，关掉能省点API额度）。

**没做的部分**（上次已经说过原因，这里不重复）：交易所净流入流出、MVRV/SOPR链上估值、
社交媒体AI情绪——这几个免费数据源做不到有意义的精度，保持诚实，没有强行凑数。

### 🔥 二级市场强势（新增）

### 🏦 一级市场融资（新增）
数据源：DefiLlama公开的融资数据。列出近期融资事件，命中知名一二线VC
（a16z、Paradigm、Binance Labs等）参投的会标⭐排在前面，作为"值得提前认识"的看好理由。

**这里要对你格外坦白一点**：这个接口不在DefiLlama官方文档列出的正式API目录里，
是他们融资页面本身在用的接口，我没办法在开发环境里直接联调验证（网络权限限制），
字段结构也没有官方保证。**如果你点开这个子标签发现一直报错或者拉不到数据，
大概率是接口的字段名或者路径变了**，把报错信息发给我，我根据实际返回的数据结构改代码；
它同时也会提示你去 https://defillama.com/raises 手动查看，不会让你两手空空。

**关于"打破信息差、提前布局"这件事，我要再泼一点冷水**：一级市场的融资信息本质上
只是"哪些机构在哪个阶段投了钱"，跟"这个项目代币未来会不会大涨"是两件不完全相关的事——
很多融资很猛的项目代币表现平平，也有没什么大牌机构背书的项目代币表现很好。这个模块
的价值是帮你**提前"认识"这些项目**，等它们未来真的上二级市场、进入"新币早期扫描"或者
"二级市场强势"这两个模块的视野时，你已经比只看K线的人多一层背景认知，仅此而已，
不代表看到融资消息就应该冲进去。
